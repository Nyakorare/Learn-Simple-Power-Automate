Introduction

Power Automate is Microsoft’s service for automating tasks across apps and services. A cloud flow reacts to triggers (email arrived, item created, schedule, button) and runs a sequence of actions with connectors.

Use it to replace repetitive copy-paste work, route approvals, sync data, and notify people—while keeping logic documented in one place.

Official overview: Get started with Power Automate.

Maker portal & environments

Open make.powerautomate.com. You will work with:

  • My flows — list of flows you own or co-own.
  • Solutions — packages for apps, flows, and connection references for ALM.
  • Environments — boundaries for data and security; use non-production environments for learning.

Types of flows

  • Automated cloud flow — starts when an event happens in a connector.
  • Instant cloud flow — user or app starts it manually (button, Power Apps, Teams).
  • Scheduled cloud flow — runs on a recurrence.
  • Desktop flow (RPA) — automates Windows/UI through Power Automate for desktop.
  • Business process flow — guided stages in model-driven apps (Dataverse).

Triggers & actions

Every flow has exactly one trigger. Each subsequent step is an action. The outputs of earlier steps appear as dynamic content for later steps.

Rename steps clearly (for example “Get manager for submitter”) so run history and handoffs stay understandable.

Variables

Use Initialize variable to create typed storage within a run. Common uses: counters, building a string across a loop, or holding parsed JSON.

Prefer scoping variables inside the smallest scope that needs them (for example inside Apply to each when appropriate).

Conditional logic

The Condition action splits yes/no paths. Switch compares one value to many cases—cleaner than nested conditions.

For parallel work, explore parallel branches; merge carefully so you do not duplicate side effects.

Loops

Apply to each walks a list (for example attachments, rows, or child items). Watch API limits and consider batching or filtering first.

Concurrency settings can speed independent iterations but can also stress downstream systems—tune deliberately.

Approvals

Approval actions pause a run until a decision is recorded in Microsoft 365. Model who approves, timeouts, and what happens on reject.

Modern approvals in Power Automate

Connectors

Connectors wrap APIs into triggers and actions. Some are standard, others premium (licensing applies). Custom connectors let your organization expose internal APIs.

Use connection references in solutions so flows can be moved between environments with correct credentials.

Expressions

Open the expression editor (fx) to transform data: dates, strings, math, and logical tests. Expressions use workflow definition language functions.

concat('Ticket-', triggerOutputs()?['body/id'])
formatDateTime(utcNow(), 'yyyy-MM-dd')

JSON

Use Parse JSON with a schema so you can reference properties as dynamic content. Generate a schema from a sample payload when prototyping.

{
  "id": 42,
  "status": "submitted"
}

HTTP & APIs

The HTTP action calls REST endpoints. Secure secrets in Azure Key Vault or environment variables as your organization requires.

For Microsoft Graph and other protected APIs, use first-party connectors or Azure AD–backed authentication patterns supported in your tenant.

Error handling

Configure Configure run after on actions to catch failures, timeouts, or skips. Wrap risky sections in a Scope and pair with a run-after branch for centralized handling.

Send meaningful notifications (including run URL) so support can replay or fix quickly.

FAQ

What is Power Automate?

Microsoft’s automation service for connecting apps and services. You design flows that start with a trigger and perform actions using connectors—often with no traditional coding.

Do I need to be a developer?

No. Many flows are built visually. Expressions and HTTP actions add power similar to lightweight development—still focused on automation, not building a full app from scratch.

Where do I practice safely?

Use a non-production environment or trial tenant. Avoid pointing experimental flows at production financial or customer systems until you have tested run history, limits, and security.
