SharePoint patterns

Microsoft Lists is built on SharePoint, so in Power Automate you usually use SharePoint triggers and actions to automate List data.

Core Microsoft Lists flow
  • Trigger: When an item is created or When an item is created or modified.
  • Get data: use Get item or Get items with Filter Query and Top Count.
  • Use data: run conditions (priority, status, due date), calculate values, or route to approvals.
  • Write back: use Update item to stamp outcomes like ApprovalStatus, Owner, or CompletedDate.
Good list design for automation
  • Use stable internal column names and keep required fields intentional.
  • Prefer metadata columns over folder-path logic where possible.
  • Store multi-step flow state with columns like ProcessingState or LastFlowRunUtc.
Common pattern: get data, shape it, then send

Example: get all open tasks due in 3 days, filter to high priority, create an HTML table, then send a Teams or email digest to owners. This helps learners connect retrieval, transformation, and delivery in one flow.
