Hands-on path

Try this in Power Automate
  • Create a flow where one action is expected to fail (for example HTTP to an invalid URL in a test env).
  • Wrap risky actions in Scope. Below it, add a parallel path and set Configure run after to has failed on the scope.
  • In the failure branch, send yourself an email that includes the run link from dynamic content.
  • Reset Configure run after to default on production flows once you understand the pattern.

Use a dev or trial environment. Designer labels change over time—use the search box in the add-action panel if a name differs slightly.
